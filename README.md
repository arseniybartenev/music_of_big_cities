# Анализ поведения пользователей Яндекс Музыки в Москве и Санкт-Петербурге

![Python](https://img.shields.io/badge/Python-3.9-3776AB?logo=python&logoColor=white)
![Pandas](https://img.shields.io/badge/Pandas-3.0.1-150458?logo=pandas&logoColor=white)

## Описание проекта

Исследование посвящено сравнению музыкальных предпочтений и поведения пользователей Яндекс Музыки в двух столицах — Москве и Санкт-Петербурге. В рамках проекта проверяются три популярных мифа:

1. Активность прослушиваний зависит от дня недели, причём в Москве и Петербурге это проявляется по-разному.
2. В понедельник утром и в пятницу вечером в городах преобладают разные жанры.
3. Москва чаще слушает поп-музыку, а Петербург — русский рэп.

**Цель исследования** — на основе реальных данных Яндекс Музыки подтвердить или опровергнуть эти гипотезы, выполнив полноценный анализ данных: от предобработки до интерпретации результатов.

## Данные

Исходные данные представлены в файле `yandex_music_project.csv` (находится в папке `data/`). Датасет содержит 65 079 записей о прослушанных треках со следующими полями:

- `userID` — идентификатор пользователя;
- `Track` — название трека;
- `artist` — имя исполнителя;
- `genre` — жанр;
- `City` — город пользователя (Moscow / Saint-Petersburg);
- `time` — время начала прослушивания;
- `Day` — день недели.

В ходе предобработки были обнаружены пропуски в столбцах `track`, `artist`, `genre`, а также явные и неявные дубликаты (например, варианты написания жанра «hiphop»). Все пропуски заполнены значением `'unknown'`, дубликаты удалены, а некорректные названия жанров приведены к единому виду.

## Этапы работы

1. **Обзор данных**  
   - Первичный анализ структуры таблицы, типов данных, количества пропусков.  
   - Вывод первых 10 строк и общей информации методом `info()`.

2. **Предобработка данных**  
   - Приведение названий столбцов к «змеиному» регистру (snake_case) и удаление лишних пробелов.  
   - Замена пропусков в столбцах `track`, `artist`, `genre` на `'unknown'`.  
   - Удаление явных дубликатов.  
   - Исправление неявных дубликатов жанра: `hip`, `hop`, `hip-hop` → `hiphop`.

3. **Проверка гипотез**  
   - **Гипотеза 1:** сравнение активности пользователей в понедельник, среду и пятницу для каждого города (функция `number_tracks`).  
   - **Гипотеза 2:** анализ топ-10 жанров утром понедельника (07:00–11:00) и вечером пятницы (17:00–23:00) с помощью функции `genre_weekday`.  
   - **Гипотеза 3:** сравнение общих жанровых предпочтений (подсчёт прослушиваний каждого жанра в Москве и Петербурге).

## Используемые технологии

- **Python** — основной язык программирования.
- **Pandas** — обработка и анализ данных (группировки, фильтрация, очистка).
- **Jupyter Notebook** — среда разработки и визуализации результатов.

## Результаты

| Гипотеза | Статус | Комментарий |
|----------|--------|-------------|
| **Гипотеза 1** (активность по дням) | Подтвердилась | В Москве пик прослушиваний — понедельник и пятница, спад в среду. В Петербурге, наоборот, среда — самый активный день. |
| **Гипотеза 2** (жанры утром/вечером) | Частично подтвердилась | В утренние часы понедельника в Москве чаще звучит жанр «world», в Петербурге — джаз и классика. Однако общая структура топ-10 схожа. Пропуски в данных (особенно в Москве) могли повлиять на точность. |
| **Гипотеза 3** (поп vs рэп) | Не подтвердилась | Поп-музыка лидирует в обоих городах. Русский рэп (rusrap) присутствует в топ-10, но его доля почти одинакова в Москве и Петербурге. |

## Выводы

Проведённый анализ показал, что различия в музыкальных предпочтениях между Москвой и Санкт-Петербургом менее выражены, чем принято считать. Основные выводы:

- Поведение пользователей действительно различается по дням недели, что подтверждает первую гипотезу.
- Временные срезы (утро понедельника / вечер пятницы) выявили лишь незначительные отличия в жанровом разнообразии, при этом значительная доля пропусков (`'unknown'`) ставит под сомнение достоверность второго вывода.
- Вопреки ожиданиям, русский рэп популярен в обоих городах примерно в равной степени, а поп-музыка остаётся безоговорочным лидером.

Таким образом, исследование демонстрирует важность тщательной предобработки данных и необходимость проверки статистических гипотез для получения объективных результатов.

## Запуск проекта

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/arseniybartenev/music_of_big_cities.git
   ```
2. Перейдите в папку проекта:
   ```bash
   cd music_of_big_cities
   ```
3. Установите зависимости:
   ```bash
   pip install pandas==3.0.1
   ```
4. Поместите файл с данными `yandex_music_project.csv` в папку `data/`.  
   *Примечание: в исходном ноутбуке используется путь `/datasets/yandex_music_project.csv`. При необходимости измените его на `data/yandex_music_project.csv` в коде.*
5. Запустите Jupyter Notebook:
   ```bash
   jupyter notebook notebooks/music_of_big_cities.ipynb
   ```
